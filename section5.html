2) A quoi sert concrètement un système de gestion de version ?

Le processus de sauvegarde et de restauration d'un site web peut être contraignant, surtout lorsque plusieurs personnes travaillent sur le même site et développent des fonctionnalités différentes qui peuvent être liées entre elles. Il est important de sauvegarder le site avant toute modification ou test afin de pouvoir le restaurer en cas de problème. Cependant, cela peut être compliqué lorsque plusieurs personnes sont impliquées dans le processus.

Le travail en équipe sur un projet informatique nécessite une organisation complexe pour éviter les conflits entre les fonctionnalités développées par chaque membre. Pour cela, il est recommandé de mettre en place un serveur distant qui contiendrait l'historique des modifications faites par chaque développeur. Git est un logiciel de versioning qui permet de coordonner le travail entre plusieurs personnes en conservant un historique des changements effectués sur des fichiers, ainsi que la coexistence de différentes versions d'un même fichier. Les développeurs peuvent ainsi accéder à l'historique des modifications pour l'intégralité du projet et savoir qui a fait ces changements.


3) Les deux modèles des logiciels de gestion de version :

modèle centralisé vs modèle décentralisé

Les logiciels de gestion de version sont basés sur deux modèles : centralisé et décentralisé. Le modèle centralisé héberge le code source sur un serveur distant, tandis que dans le modèle décentralisé, chaque utilisateur télécharge et héberge le code source sur sa propre machine. Git a popularisé le modèle décentralisé qui offre des avantages tels que la simplicité et la flexibilité du travail, ainsi que la sécurité grâce à la possibilité d'utiliser une copie du projet comme backup en cas de corruption du serveur central.


4) Qu’est-ce que GitHub ?


Git est un logiciel de gestion de version qui permet d'enregistrer les modifications effectuées sur un projet et de revenir à une version précédente. Un dépôt est la copie de l'intégralité des fichiers d'un projet et de leur version située sur le serveur central. GitHub est un service en ligne qui permet d'héberger des dépôts Git, et c'est le plus grand hébergeur de dépôts Git du monde. Les dépôts hébergés sur GitHub sont souvent publics, ce qui signifie que n'importe qui peut télécharger le code et contribuer à leur développement en proposant de nouvelles fonctionnalités.

10) les etats des fichiers: 


On dit qu'un fichier dans git est suivi s'il est enregistre en base et non-suivi s'il n'appartenait pas au dernier instantane .

les fichiers sont tous non suivi en damarant un depotgit en important un repertoire deja existant et ils sont tous suivi lorsqu'on clone un depot deja existant .

chaque fichier suivi peut avoir l'etat : soit "valide" lorsqu'il est stockee dans la base de donnees ou lors du clonage d'un depot existant, ou "modifie" tant qu'il n'a pas ete indexe, et finalement "indexe"  lorsqu’on indique à Git que le fichier modifié ou que 

le nouveau fichier doit faire partie du prochain instantané dans sa version actuelle.

 

Enfin,lorsqu’on demande à Git de prendre l’instantané , les fichiers faisant partie a ce dernier sont consideres "valides".
